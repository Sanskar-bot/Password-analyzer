Installation

Up and running in under two minutes

Chrome Web Store submission is on the roadmap. For now, follow these steps to load PassGuard as a developer extension.

1
Download the Extension Package
Click the Download button above to download the extension as a ZIP file. No installer or build step required.
passguard-extension-v1.0.0.zip
2
Extract the ZIP File
Right-click the downloaded file and select "Extract All" (Windows) or double-click (macOS). Note the folder location — you will need it in Step 7.
The extracted folder should be named passguard-extension-v1.0.0 and contain a manifest.json file at its root.
3
Open Chrome (or Edge / Brave)
Open your Chromium-based browser. PassGuard supports Chrome, Edge, Brave, and Opera — all using the same extension package.
4
Navigate to Extensions
In the address bar, type the extensions URL and press Enter:
chrome://extensions
For Edge: edge://extensions — for Brave: brave://extensions
5
Enable Developer Mode
In the top-right corner of the Extensions page, toggle Developer mode to ON. New buttons will appear in the top-left.
Developer mode is required to load extensions that are not from the Chrome Web Store. It does not expose any security risk.
6
Click "Load Unpacked"
Click the Load unpacked button that appeared after enabling Developer mode. A file picker dialog will open.
7
Select the PassGuard Folder
Navigate to the extracted folder from Step 2. Select the folder that contains the manifest.json file and click Select Folder.
Select the folder itself, not the manifest.json file inside it.
Extension Installed
PassGuard now appears in your extensions list. Navigate to any website with a password field and the analyzer widget will appear automatically below the field.

Architecture

How PassGuard works inline

A six-stage analysis pipeline runs locally in your browser every time you type into a password field.

Password Field Detected
MutationObserver watches every page for <input type="password"> fields, including dynamically injected forms.
Context Classification
Determines whether you are on a login form or a signup/account-creation form. Generator only shown during signup.
Multi-Layer Analysis
Six scoring modules run in parallel: entropy, patterns, wordlist matching, character variety, brute-force simulation, and username similarity.
Personalized Attack Simulation
If a Security Profile is saved, a targeted dictionary is generated from your personal information and tested against your current password.
Composite Score
A weighted composite security score from 0–100 is calculated, with individual component scores shown in the breakdown.
Actionable Recommendations
Specific, contextual suggestions displayed inline — not generic advice, but recommendations based on detected issues.

Technical Details

For the technically curious

Architecture and implementation details, kept collapsed to avoid overwhelming non-technical users.

PassGuard uses Manifest V3, the current standard for Chrome extensions. MV3 improves security over MV2 by restricting remote code execution and requiring declarative content rules.

  • Chrome 88+
  • Edge 88+
  • Brave (latest)
  • Opera 74+

Firefox support is in progress — the MV3 API surface is now nearly compatible. A Firefox build is planned for v1.1.

The extension uses localStorage with the key pg_profile_v1 to store your Security Profile. This data:

  • Never leaves the browser
  • Is not synced via chrome.storage.sync
  • Can be exported as JSON from the Profile page
  • Can be cleared at any time from the Profile page or browser settings

All other analysis data (dictionaries, scores, entropy) is computed in-memory and discarded after each keystroke. Nothing is persisted.

The PassGuard widget is injected into a Shadow DOM host element attached to the page. This means:

  • PassGuard's CSS cannot affect the host page's styles
  • The host page's CSS cannot affect PassGuard's styles
  • PassGuard's JavaScript runs in the content script context, isolated from page scripts
  • The widget cannot read or write to the host page's JavaScript state

PassGuard is 100% offline. All analysis modules, wordlists, and generation logic are bundled into the extension package. There are no CDN imports, no API calls, and no remote resources.

The extension functions identically with or without an internet connection.

The complete source code is available at github.com/Sanskar-bot/Password-analyzer.

The extension source is in the extension/ directory. Key files:

  • extension/manifest.json — permissions, version, entry points
  • extension/content/content.js — main content script
  • extension/modules/ — analysis modules
  • extension/popup/ — extension popup UI
